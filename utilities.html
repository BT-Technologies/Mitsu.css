Utilities

Utility classes let you compose layout and spacing quickly without creating one-off CSS.

Use utilities for speed and consistency. Reserve custom CSS for patterns that cannot be composed from existing primitives.

Text Links (Professional States)

Use .lnk for production-ready text links with clear normal, hover, visited, active (click), and focus-visible states.

.lnk.lnk-sub.lnk-quiet.lnk-off.lnk-h.lnk-v.lnk-a

Normal: .lnk

Helper: default state for body links and inline documentation references.

Hover preview: .lnk:hover

Helper: hover increases clarity with stronger underline/color contrast.

Visited preview: .lnk:visited

Helper: visited tone helps users track what they already opened.

Active preview: .lnk:active

Helper: active/click state gives immediate press feedback.

Subtle variant: .lnk-sub

Helper: use subtle links inside dense cards and metadata blocks.

Quiet variant: .lnk-quiet

Helper: quiet links stay clean with no hover visual changes, ideal in busy toolbars.

Inactive: .lnk-off

Helper: .lnk-off shows disabled state (faded color) and blocks clicks. Links with data-vstat display a visited status after click.

Text Columns (1 to 5)

Split long content into newspaper-like columns. Best for references, changelog notes, and dense documentation sections.

.text-cols-1 ... .text-cols-5.text-col-gap-sm/md/lg.text-col-rule

MitsuCSS keeps utility classes predictable so teams can scan layout quickly and make safe changes without hunting through large custom stylesheets.

Column utilities work on parent containers and flow child paragraphs naturally, helping large blocks become easier to read on wide screens.

For responsive docs, switch from three columns to one or two columns with breakpoint-specific wrappers in your project layer if needed.
