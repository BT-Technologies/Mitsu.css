Cards

Cards are surface containers for grouped content. This demo includes the 4 production card patterns and reveal/flip behavior.

When to use: Use a card when content belongs to one idea, action, or media item.

Classes: .mitsu-card + demo helpers .cards-demo-* for presentation patterns.

1. Normal Card

Title and body only. Best for short summaries, metrics, and compact explanations.

Helper: keep this variant clean, with one clear message and no media noise.

2. Image Card

Title, image, and body. Good for product previews, docs highlights, and release notes.

Helper: media first, then concise text. Keep image ratio stable across a list.

3. Action Card

Title, body, optional image, and action controls for direct task completion.

Read details

Helper: use one main CTA and one secondary link only.

4. Flip Image Card

Helper: this pattern is interactive; keep back text short and action-focused.

Helper per item: reveal animation runs on load; flip card toggles front/back on click and keyboard.

Buttons

Select ? on any example to see how to use it.

Tones: primary, semantic states, and support actions.

Use .mitsu-btn for the main action of a section or form. Use .mitsu-btn-info for details/help actions. Use .mitsu-btn-success for save/confirm actions. Use .mitsu-btn-warning before risky but reversible actions. Use .mitsu-btn-danger for destructive/irreversible actions. Use .mitsu-btn-secondary for support actions next to primary. Use .mitsu-btn-ghost for low-emphasis actions.

Sizes: compact to large CTA scale.

Use .mitsu-btn-sm in dense UIs like tables and toolbars. Use .mitsu-btn-md as default balanced size. Use .mitsu-btn-lg for key CTA and touch-first targets.

Icons and badges: quick recognition plus status counters.

Use .mitsu-btn-icon with a text label for better recognition. Icon-only buttons need aria-label and should map to common actions. Use .mitsu-btn-badge for counts or state in action buttons. xLink Button 9Use an anchor with .mitsu-btn when action must navigate.

Outline styles: lighter emphasis and high-attention double outline.

Use .mitsu-btn-outline for secondary emphasis. Keep outline tone aligned with the action intent. Use .mitsu-btn-double-outline only for high-attention cases. Double outline works best when danger action must stand out.

Button groups: segmented controls for related actions.

Use .mitsu-btn-group for tightly related options.
Action groups should follow real workflow order.
Icon groups are best for quick state transitions: cancel, warn, done.

Code Assist: Create A Dialog

Use one snippet below as a quick starting point.

1. Top-Down (from parent)

<button class="mitsu-btn" data-mitsu-toggle="#my-drop">Open</button>
<div class="dialog-drop-parent">
  <div id="my-drop" class="dialog-drop mitsu-hidden">...</div>
</div>

2. Modal (outside click close)

<button class="mitsu-btn" data-mitsu-toggle="#my-modal">Open Modal</button>
<div id="my-modal" class="mitsu-dialog mitsu-hidden" data-mitsu-outside-close="true">
  <div class="mitsu-dialog-card">...</div>
</div>

3. Classic dialog

<button class="mitsu-btn" data-mitsu-toggle="#my-dialog">Open Dialog</button>
<div id="my-dialog" class="mitsu-dialog mitsu-hidden">
  <div class="mitsu-dialog-card">...</div>
</div>

4. Window message (ask/input)

<button class="mitsu-btn" data-mitsu-toggle="#my-window">Open Window</button>
<div id="my-window" class="mitsu-dialog mitsu-hidden dialog-window-shell">
  <div class="mitsu-dialog-card dialog-window-card">...input...</div>
</div>

Forms

Use label/input pairs consistently for accessibility. Keep field groups short and aligned with user tasks.

When to use: Data capture fields inside forms, dialogs, and settings pages.

Classes: .mitsu-label, .mitsu-input. Pair every input with a visible label.

Helper per item: always pair each input with a label and meaningful placeholder example.
