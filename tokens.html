Design Tokens

Base variables from css/mitsu.css drive colors, spacing, radius, and shadows.

Use these tokens as your source of truth. When branding changes, update tokens first so every component inherits the new style automatically.

Utility naming: use color-* for foreground text and bgr-* for background color.

Helper per item: each swatch item shows visual preview + exact token variable for copy/paste.

Radius and Shadow

Radius controls edge softness and shadow controls depth. Use smaller radius for compact controls and larger radius for cards/dialog surfaces.

Note: CSS does not support negative border-radius. MitsuCSS includes cut-corner utilities (.mitsu-radius-cut-sm, .mitsu-radius-cut-md) as a safe negative-like style.

Cut radius helper: .mitsu-radius-cut-sm and .mitsu-radius-cut-md create chamfered corners with matching shaped borders.

Customize: --mitsu-cut-size, --mitsu-cut-border-width, --mitsu-cut-border-color, --mitsu-cut-fill.
